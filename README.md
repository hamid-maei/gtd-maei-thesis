# gtd_phd_maei — Maei (2011) PhD thesis Fig 5.1 (gradient TD)

A clean, OOP reproduction of Figure 5.1 of Maei's 2011 PhD thesis (the
corrected version of Sutton, Maei, et al., ICML 2009). The numerics are
identical to the reference implementation in
`archive/Fast_Gradient_TD_Codes_ICML09paper/python_codes/` — this package bundles the same
algorithms behind a small, fully testable object model.

## Layout

```
gtd_phd_maei/
  environments.py       Environment ABC, RandomWalkEnv, BoyanChainEnv
  features.py           FeatureMap ABC + Tabular / Dependent / Inverted / BoyanTriangle
  algorithms.py         LinearTDAlgorithm ABC + TD0 / TDC / GTD2 / GTD1
  agents.py             LinearTDAgent (composes algorithm + feature map + step sizes)
  evaluation.py         RMSPBEEvaluator (closed-form A, b, C, inv(C))
  training.py           Trainer (one episode) + ParameterSweep + SweepResult
  figure_51_config.py   Canonical 100 runs + per-problem episode counts
  reproduce_figure_51.py  One command: full-scale ``run_all.py`` (thesis numerics)
  plot_common.py        Shared plotting helpers (param study + best curves)
  plot_thesis_figure_51.py
  randwalk_tabular_exp.py
  randwalk_dependent_exp.py
  randwalk_inverted_exp.py
  boyan_exp.py
  run_all.py            End-to-end driver (experiments + plots)
  tests/                pytest-style unit + consistency tests
  data/                 Generated by experiments
  figures/              Generated by plotting
```

## Quick start

Thesis figure 5.1 is defined by the **saved RMSPBE tensors** in
`data/*/results.npz` (100 runs; 200 / 400 / 500 / 100 episodes). PNG/PDF
under `figures/` are only thesis-consistent after you regenerate that
data at full scale—not after `--quick`.

```bash
cd gtd_phd_maei

# Full thesis reproduction (recommended)
PYTHONPATH=. python reproduce_figure_51.py
# equivalent:
PYTHONPATH=. python run_all.py

# Smoke test only (5×20); curves will not match the thesis
PYTHONPATH=. python run_all.py --quick

# Run the test suite
PYTHONPATH=. python -m pytest tests -q
```

## Architecture

```
Environment ----obs, reward, done----> Agent
FeatureMap  ----phi(s)--------------> Agent
Agent       ----phi, phi', r, alpha, beta-> LinearTDAlgorithm
LinearTDAlgorithm ----theta--------> RMSPBEEvaluator
RMSPBEEvaluator   ----RMSPBE-------> ParameterSweep
```

The `Agent` is the only object that knows about features. The
`Environment` returns plain state indices; the `FeatureMap` turns those
into feature vectors. Algorithms receive `phi`, `phi'`, `reward` only.
This matches the OAK split described in `CLAUDE.md` at the project root.

## Algorithms (Maei, 2011, Chapter 5)

| Class  | Update                                                                            |
| ------ | --------------------------------------------------------------------------------- |
| `TD0`  | `theta += alpha * delta * phi`                                                    |
| `TDC`  | `theta += alpha*(delta*phi - gamma*phi'*(w.phi)); w += beta*(delta - w.phi)*phi`  |
| `GTD2` | `theta += alpha*(phi - gamma*phi')*(w.phi);       w += beta*(delta - w.phi)*phi` |
| `GTD1` | `theta += alpha*(phi - gamma*phi')*(w.phi);       w += beta*(delta*phi - w)`      |

`delta = reward + gamma * (theta . phi') - (theta . phi)`.

`eta = beta / alpha` is the secondary step-size ratio; `eta = 0`
collapses TDC / GTD2 / GTD1 to plain TD(0) (because `w` stays zero).

## Reproducing Figure 5.1

`PYTHONPATH=. python reproduce_figure_51.py` (or `python run_all.py` with
no `--quick`) produces:

* `data/<problem>/results.npz` for each of the four problems.
* `figures/<problem>/*.png` and matching `*.pdf` per-problem panels (vector
  PDFs open cleanly in Preview / Acrobat).
* `figures/thesis_figure_5_1.png` and `figures/thesis_figure_5_1.pdf` — 2×4
  combined figure laid out like the thesis.

Sweep grids match the thesis exactly:

| Problem       | runs | episodes | alpha grid                                | eta grid               |
| ------------- | ---- | -------- | ----------------------------------------- | ---------------------- |
| RW tabular    | 100  | 200      | {.03, .06, .12, .25, .5}                  | {1/4, 1/2, 1, 2}       |
| RW dependent  | 100  | 400      | {.008, .015, .03, .06, .12, .25, .5}      | {1/4, 1/2, 1, 2}       |
| RW inverted   | 100  | 500      | {.03, .06, .12, .25, .5}                  | {1/4, 1/2, 1, 2}       |
| Boyan chain   | 100  | 100      | {.015, .03, .06, .12, .25, .5, 1, 2}      | {1/8, 1/4, 1/2, 1, 2}  |

`eta = 0` is also swept so that the `eta = 0` row of the TDC results
serves as the TD(0) baseline (cf. thesis p. 49, footnote 3).

## Numerical equivalence

`tests/test_consistency.py` runs each problem at `runs=2, episodes=20,
seed=0` against the reference scripts under
`archive/Fast_Gradient_TD_Codes_ICML09paper/python_codes/` and asserts
`np.allclose(new, ref, atol=1e-12)` on every RMSPBE array. Bit identical
because:

1. Same RNG seed.
2. Same sweep traversal order.
3. Same algorithm-update order at each step (TDC, GTD2, GTD1).
4. Same `phi' = 0` convention on terminal transitions.
